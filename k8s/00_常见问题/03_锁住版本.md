## 1. 核心包锁定 (apt-mark)

这是最重要的一步，防止 `apt upgrade` 误伤关键组件。

## 锁定内核 (Kernel)

内核升级是导致 NVIDIA 驱动失效和 K8s 节点不稳定的首要原因。

```bash
# 锁定通用内核和头文件
sudo apt-mark hold linux-image-generic linux-headers-generic linux-generic
```

## 锁定 NVIDIA 驱动与工具链

防止驱动版本跳变导致 Kit 挂起或 K8s GPU 插件失效

```Bash
# 根据你当前的驱动版本锁定（例如 535）
sudo apt-mark hold nvidia-driver-535 libnvidia-compute-535 nvidia-dkms-535
# 锁定容器运行时（这对 K8s 极其重要）
sudo apt-mark hold nvidia-container-toolkit nvidia-container-runtime
```

## 锁定容器运行时与 K8s 组件

防止 CRI (Container Runtime Interface) 升级导致集群重置。

```bash
sudo apt-mark hold containerd.io kubelet kubeadm kubectl
```
