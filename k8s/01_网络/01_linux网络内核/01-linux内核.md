# linux内核

## linux内核基础概念

### 1. 内核的设计哲学：上帝视角与中介

- **内核 (Kernel)**：硬件的唯一管理者（大管家）。
- **核心职能**：管理 CPU（调度）、内存（分配）、文件系统（存储）、网络栈（通信）。
- **一切皆文件**：Linux 将所有资源（硬件、网络、进程、空间）都抽象化为“文件”，提供统一的操作接口。

### 2. 核心概念：文件句柄 (File Descriptor, FD)

- **定义**：一个非负整数，可以理解为**内核对象的“受控指针”**。
- **逻辑链条**：
  1. **应用层**：申请资源（如 `socket()`） -> 拿到数字存根（句柄）。
  2. **内核层**：维护一张“句柄表”，记录数字对应的真实对象地址。
  3. **操作**：应用层对句柄执行 `read/write` -> 内核通过句柄找到对象 -> 驱动程序操作硬件。
- **意义**：建立安全屏障。应用层拿不到真实物理地址，只能通过句柄请求内核办事。

### 3. Namespace (NS)：视图隔离

- **本质**：内核在全局账本之外开辟的**“虚拟分账”**（VR眼镜）。

- **Network Namespace**：隔离了网络栈（网卡、路由表、防火墙规则、端口）。

- **与句柄的关系**：每个 Namespace 在内核中也是一个对象，拥有对应的文件路径和句柄。

  

## namespace的基础概念

###  1. 什么是 Namespace？（本质与错觉）

Namespace 的作用是**“隔离内核资源”** 。它将全局系统资源（如 IP 地址、进程 ID 等）分割并装入独立的抽象空间 。它给其中的进程造成了两个关键错觉：

- **唯一性**：进程认为自己是系统里唯一的进程 。

- **独占性**：进程认为自己独享系统的所有资源 。



### 2. 六类 Namespace 的作用

详细列出了 Linux 内核实现的 6 种 Namespace，每一类隔离不同的资源 ：

| **Namespace 类型** | **隔离的资源**                       | **在容器/K8s中的直观体现**                          |
| ------------------ | ------------------------------------ | --------------------------------------------------- |
| **Network (net)**  | 网络设备、IP、端口、路由表、防火墙 8 | 每个 Pod 拥有独立的 IP 和端口范围，互不干扰 9。     |
| **PID**            | 进程 ID 数字空间                     | 容器内看到自己的初始进程 PID 为 1 10101010。        |
| **Mount (mnt)**    | 文件系统挂载点                       | 容器拥有独立的文件系统视图，看不到宿主机的文件 11。 |
| **UTS**            | 主机名和 NIS 域名                    | 容器可以有独立的主机名（hostname）12。              |
| **IPC**            | 进程间通信（信号量、消息队列等）     | 跨容器的进程无法直接通过 IPC 通信 13。              |
| **User**           | 用户 ID 和组 ID 数字空间             | 容器内的 Root 用户在宿主机看来可能只是普通用户 14。 |

### 3. Namespace 与内核的交互机制

- **默认状态与初始空间**：默认情况下，所有进程都处在和宿主机相同的“初始根 Namespace”里，享有全局资源 。

- **API 操作（三把手术刀）**：进程通过特定的系统调用（API）请求内核进行空间切换或创建 ：

  - **`clone()`**：创建新进程时，通过参数（如 `CLONE_NEWNET`）告诉内核：“请为这个新进程装修一个独立的房间（新 NS）” 。

  - **`setns()`**：将一个现有进程“扔进”已经存在的房间里。这是进入容器内部（如 `nsenter` 命令）的底层原理 。

  - **`unshare()`**：让当前进程跳出原来的大厅，原地创建一个新房间并搬进去 。

- **文件句柄与 `/proc` 目录**：内核为每个进程在 `/proc/[PID]/ns/` 目录下生成一组指向不同 NS 的文件 。**句柄（File Descriptor）**就像是进入这些房间的钥匙 。只要打开了这些文件，对应的 Namespace 就会被内核保留，即使里面没有进程运行 。
