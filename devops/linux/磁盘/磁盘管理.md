## raid

### RAID 的目的和优势

- **数据冗余**：RAID 的核心目的是提供数据冗余（如 RAID 1、RAID 5、RAID 6），从而在硬盘出现故障时防止数据丢失。
- **性能提升**：通过条带化（如 RAID 0、RAID 10），RAID 还可以提高数据读写速度。
- **RAID 通常在物理层面操作**：它处理的是如何在物理硬盘上组织数据。



### RAID 常见级别：

- **RAID 0 (条带化)**：将数据分割成块，平均分布到两个或更多的磁盘上，没有冗余，提高读写速度，但任一磁盘故障会导致所有数据丢失。
- **RAID 1 (镜像)**：完整复制数据到两个或更多磁盘上，提供数据冗余，如果一块磁盘失败，其他磁盘可以继续提供数据。
- **RAID 5 (带奇偶校验的条带化)**：数据和奇偶校验信息被分布到三个或更多的磁盘上，单磁盘故障时，可以重建数据。
- **RAID 10 (镜像和条带化的组合)**：结合 RAID 0 和 RAID 1 的特点，至少需要四个磁盘，提供较好的性能和冗余。

### RAID 的使用

**创建 RAID**：使用 `mdadm` 命令创建 RAID 阵列。例如，创建一个 RAID 1 阵列可以使用如下命令：

```bash
sudo mdadm --create --verbose /dev/md0 --level=1 --raid-devices=2 /dev/sda1 /dev/sdb1
```

这里，`/dev/md0` 是创建的 RAID 设备，`--level=1` 指定 RAID 级别为 1，`/dev/sda1` 和 `/dev/sdb1` 是用于 RAID 的分区。



## lvm

### LVM 的目的和优势

- **存储灵活性**：LVM 允许动态调整逻辑卷的大小，可以在不丢失数据的情况下增加或减少存储空间。
- **逻辑管理**：LVM 作用于逻辑层面，可以跨多个物理磁盘或RAID阵列创建逻辑卷。
- **快照和克隆**：LVM 支持快照功能，这对于备份或恢复特定时间点的数据非常有用



### LVM 主要组件：

- **物理卷 (PV)**：物理硬盘或其分区，用作LVM的基础。
- **卷组 (VG)**：一组物理卷，形成一个存储池。
- **逻辑卷 (LV)**：从卷组中分配的存储空间，可以被格式化和挂载使用。

#### 如何使用 LVM：

1. **创建物理卷**：使用 `pvcreate` 命令将物理硬盘或分区初始化为物理卷。
2. **创建卷组**：使用 `vgcreate` 命令创建卷组，可以添加一个或多个物理卷到卷组中。
3. **创建逻辑卷**：使用 `lvcreate` 命令从卷组中分配空间创建逻辑卷。
4. **格式化和挂载逻辑卷**：可以将逻辑卷格式化为任何文件系统并挂载使用，就像普通的磁盘分区一样。



### lvm的使用

要检查在 Linux 系统中某个磁盘（如 sdc）是否被用作 LVM 的一部分，可以使用以下方法：

#### 查看命令

##### 1. 查看物理卷信息

运行 `pvdisplay` 命令可以显示系统中所有物理卷（PV）的详细信息。要查看特定磁盘的信息，可以直接指定磁盘设备，例如：

```bash
sudo pvdisplay /dev/sdc
```

如果 `/dev/sdc` 被用作 LVM 的物理卷，这个命令会显示其详细信息，包括它属于哪个卷组（VG）。

##### 2. 查看卷组信息

运行 `vgdisplay` 命令可以查看所有卷组（VG）的信息。这有助于理解物理卷是如何组织成卷组的：

```bash
sudo vgdisplay
```

这个命令会列出系统中所有的卷组及其包含的物理卷。



#### 创建命令

1. **创建物理卷 (PV)**： 将 磁盘设备 `/dev/sdb` 初始化为物理卷：

   ```
   sudo pvcreate /dev/sdb
   ```

2. **创建卷组 (VG)**： 创建一个新的卷组，并添加物理卷到其中：

   ```bash
   sudo vgcreate vgdata /dev/sdb
   ```

3. **创建逻辑卷 (LV)**： 在卷组内创建逻辑卷：

   ```bash
   sudo lvcreate -l 100%FREE -n lvdata vgdata
   ```

4. **格式化逻辑卷**： 假设你要使用 ext4 文件系统：

   ```shell
   sudo mkfs.ext4 /dev/vgdata/lvdata
   ```



#### 删除命令

在 LVM 中移除物理卷（磁盘）可能导致数据丢失，这取决于该物理卷上是否存储了逻辑卷的部分数据以及是否有足够的空间在其他物理卷上重新分布这些数据。处理 LVM 和物理卷时必须非常小心。下面是如果你需要从 LVM 卷组中移除物理卷（磁盘）应该遵循的步骤：

##### 确认磁盘使用情况

1. **检查物理卷的使用情况**：使用 `pvs` 或 `pvdisplay` 命令查看每个物理卷的使用情况。

##### 移动数据（如果需要）

1. **如果物理卷正在使用**，你需要先将该物理卷上的数据移动到卷组内的其他物理卷上。可以使用 `pvmove` 命令来实现：

   ```bash
   sudo pvmove /dev/sdx
   ```

   这里 `/dev/sdx` 是你想要移除的物理卷。`pvmove` 会将 `/dev/sdx` 上的数据移动到同一卷组中的其他物理卷上。

##### 从卷组中移除物理卷

1. **使用 `vgreduce` 移除物理卷**：一旦该物理卷上没有数据，你可以从卷组中移除它：

   ```
   sudo vgreduce vg_name /dev/sdx
   ```

替换 `vg_name` 为你的卷组名称，`/dev/sdx` 为要移除的物理卷。



## raid 和 lvm 的区别

```powershell
RAID 主要用于提高数据冗余和性能，而 LVM 提供了磁盘容量的灵活管理。这两者在现代存储管理中互补使用，以实现高效和灵活的数据存储解决方案
```



## raid 和 lvm 的混合使用

#### 业务场景：

要将三个磁盘 sdc、sdd、sde 组成 RAID 0 并使用 LVM 挂载到 `/data_hdd` 目录下，你可以按照以下步骤进行操作：

#### 1. 创建 RAID 0

1. **安装必要的工具**： 如果尚未安装 `mdadm`，需要先安装它。在大多数 Linux 发行版中，可以使用包管理器安装：

   ```bash
   sudo apt-get install mdadm  # Debian/Ubuntu
   sudo yum install mdadm      # CentOS/RHEL
   sudo dnf install mdadm      # Fedora
   ```

2. **创建 RAID 0 阵列**： 使用 `mdadm` 创建一个包含 sdc、sdd 和 sde 的 RAID 0 阵列：

   ```bash
   sudo mdadm --create --verbose /dev/md0 --level=0 --raid-devices=3 /dev/sdc /dev/sdd /dev/sde
   ```

   这会创建一个名为 `/dev/md0` 的设备，作为 RAID 阵列。

#### 2. 创建文件系统

在 RAID 阵列上创建一个文件系统，比如使用 ext4：

```bash
sudo mkfs.ext4 /dev/md0
```

#### 3. 配置 LVM

1. **创建物理卷 (PV)**： 将 RAID 阵列设备 `/dev/md0` 初始化为物理卷：

   ```bash
   sudo pvcreate /dev/md0
   ```

2. **创建卷组 (VG)**： 创建一个新的卷组，并添加物理卷到其中：

   ```bash
   sudo vgcreate vgdata /dev/md0
   ```

3. **创建逻辑卷 (LV)**： 在卷组内创建逻辑卷：

   ```bash
   sudo lvcreate -l 100%FREE -n lvdata vgdata
   ```

#### 4. 格式化和挂载逻辑卷

1. **格式化逻辑卷**： 假设你要使用 ext4 文件系统：

   ```bash
   sudo mkfs.ext4 /dev/vgdata/lvdata
   ```

2. **挂载逻辑卷**： 将逻辑卷挂载到 `/data_hdd` 目录：

   ```bash
   sudo mount /dev/vgdata/lvdata /data_hdd
   ```

3. **确保挂载在重启后保持**： 编辑 `/etc/fstab` 文件，添加如下行以在系统启动时自动挂载逻辑卷：

   ```bash
   /dev/vgdata/lvdata /data_hdd ext4 defaults 0 2
   ```

完成这些步骤后，你将有一个基于三个磁盘的 RAID 0 阵列，这个阵列被配置为 LVM 的一部分，并挂载到 `/data_hdd` 目录下。这样，你就可以利用所有磁盘的存储空间，并通过 LVM 管理它们了。
